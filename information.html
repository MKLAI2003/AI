Step 1: Data Collection

Data Collection

Data were downloaded from CCTX library, an open-source digital currency trading framework, it encapsulates most of the exchange APIs in the world.
We are choosing API from Binance, a global cryptocurrency exchange, for the data resource.

We were collected in the pandas dataframe format by using python programming language, which gives us the price variation in a given time period in terms of Open, Close, High, and Low Price, Volume of bitcoin.

The assets were all those OCHLV of BTC/USDT from 17 August, 2017 to 29 November, 2023 , In Figure on the right side, We obtained of 2296 sample with the distribution.

Step 2: Feature Extraction

Feature Extraction

Feature extraction is important for machine learning models in cryptocurrency trading.

Technical indicators like EMA crossovers and Z-Scores help detect market behavior and determine investment worthiness.

The model was trained using EMA crossovers and WMA150, and later added Z-Scores to improve accuracy.

The accuracy score was then evaluated.

Step 3: Data Preprocessing

Data Preprocessing

Calculate the value of EMA5, EMA30 and WMA150

EMA5 will take the target of 5 previous days.
EMA30 takes 30 the previous day.
WMA150 is 150 the previous day and is close to the variable x, so we predict the target y.
Bullish=1
Bearish=0

Whether the market is bullish or bearish, if the close price is larger than WMA150 and EMA5 larger than EMA30, it should be bullish. Otherwise, it should be bearish.

Next, all incorrect data will be removed from the dataset during data cleaning and deleted 'nan' values. We created a new column called "predict_Y" based on x.

Lastly, feature scaling was applied to normalize EMA5, EMA30, and WMA150 to prevent bias and ensure that all features contribute equally.

Step 4: Model Training

Model Training

Random Forest(RF) Model:
The algorithm will create multiple decision trees using the training dataset.
Each decision tree is built independently of the others.
New input is passed through each decision tree and processed from the root node down based on the conditions specified by the features at each node.
The final prediction is determined by aggregating the predictions from all the decision trees.
The majority prediction is selected as the final result.
The result of our model is shown on the right side.

Long-Short Term Machine(LSTM) Model:
This model shows the price to users
LSTM is a neural network that predicts stock market prices.
2-layers LSTM model is used for the project.
The model uses 100 days of stock prices to predict the next lookup time step of 15 days.
The training and testing sets are split by date.
The result shows that LSTM has high trend prediction accuracy, which helps users make decisions for trading operations.

Support Vector Machine(SVM) Model:
This model tells user buy or not
SVM model can be used for profit prediction in trade trend.
SVM provides a more concise view of trading operations compared to LSTM.
SVM algorithm is not suitable for large data sets, but still provides good results.
The model has an accuracy of around 50% in test data.
The buy or sell signals, daily returns, strategy returns, cumulative and strategy cumulative returns can be predicted.
The model did a good job overall in predicting the buy or sell signals and the returns.

With AI training, the Sharpe ratio of a strategy can be improved.

A higher Sharpe ratio indicates better risk-adjusted performance.
